¡Zarpamos! 🚢 Pasamos al famosísimo dataset del **Titanic**, un clásico indispensable en el análisis de datos para entender tasas de supervivencia según la clase, el género o la edad.

**1. Muestra toda la información de todos los pasajeros de la tabla.**

SQL

```sql
SELECT *
FROM titanic;
```

**Explicación:** Extrae absolutamente todas las columnas y filas de la tabla `titanic`. 
📝 **Chuleta:** `SELECT *` es ideal para un vistazo rápido, pero en la vida real siempre es mejor especificar el nombre exacto de las columnas que necesitas para ahorrar memoria.

**2. Muestra únicamente los nombres de las ciudades de embarque que aparecen en la tabla.**

SQL

```sql
SELECT DISTINCT ciudad_embarque
FROM titanic;
```

**Explicación:** Proyecta la columna de la ciudad, pero elimina los duplicados para que cada ciudad aparezca solo una vez. 
📝 **Chuleta:** `DISTINCT` es la herramienta definitiva para descubrir "qué opciones únicas existen" en una columna categórica.

**3. Selecciona únicamente la columna de la edad de todos los pasajeros.**

SQL

```sql
SELECT edad
FROM titanic;
```

**Explicación:** Proyección básica de una sola columna numérica. 
📝 **Chuleta:** El resultado tendrá tantas filas como pasajeros haya en el barco, incluyendo aquellas donde la edad esté vacía.

**4. Muestra el género y la edad de cada pasajero.**

SQL

```sql
SELECT genero, edad
FROM titanic;
```

**Explicación:** Proyección de múltiples columnas. 
📝 **Chuleta:** Las columnas en el `SELECT` siempre deben ir separadas por comas. El orden en el que las escribas es el orden en el que aparecerán en pantalla.

**5. Muestra la clase numérica y el precio de la tarifa pagada por cada pasajero.**

SQL

```sql
SELECT clase, tarifa
FROM titanic;
```

**Explicación:** Igual que la anterior, extraemos los datos económicos y de estatus de los pasajeros.

**6. Lista el identificador del pasajero, su género y si sobrevivió o no.**

SQL

```sql
SELECT
  id,
  genero,
  CASE WHEN superviviente = 1 THEN 'sobrevivio' ELSE 'no' END AS sobrevivio
FROM titanic;
```

**Explicación:** Transforma un dato numérico (`1` o `0`) en un texto legible (`'sobrevivio'` o `'no'`) al vuelo usando un condicional. 
📝 **Chuleta:** `CASE WHEN ... THEN ... ELSE ... END` es el equivalente a un bloque "If-Else" en programación. Acuérdate de cerrarlo siempre con la palabra `END`.

**7. Muestra las columnas de clase literal (texto) y ciudad de embarque.**

SQL

```sql
SELECT clase_literal, ciudad_embarque
FROM titanic;
```

**Explicación:** Proyección simple de dos columnas de texto.

**8. Muestra todos los datos de los pasajeros que tienen exactamente 30 años.**

SQL

```sql
SELECT *
FROM titanic
WHERE edad = 30;
```

**Explicación:** Filtrado numérico exacto usando la cláusula `WHERE`. 
📝 **Chuleta:** Los números no necesitan comillas simples a su alrededor, a diferencia del texto.

**9. Selecciona los pasajeros que viajaban en la clase 1.**

SQL

```sql
SELECT *
FROM titanic
WHERE clase = 1;
```

**Explicación:** Filtra las filas evaluando si el valor de la columna `clase` es igual a 1.

**10. Muestra los datos de los pasajeros que embarcaron en 'Southampton'.**

SQL

```sql
SELECT *
FROM titanic
WHERE ciudad_embarque = 'Southampton';
```

**Explicación:** Filtrado de texto exacto. Solo mostrará a los que subieron en ese puerto. 
📝 **Chuleta:** En SQL, el texto SIEMPRE va envuelto en comillas simples (`' '`). Y recuerda que es sensible a mayúsculas: `'Southampton'` no es lo mismo que `'southampton'`.

**11. Lista los pasajeros de género femenino.**

SQL


**Explicación:** Igual que el anterior, filtramos por una cadena de texto.

**12. Muestra los pasajeros que pagaron una tarifa igual a 0.**

SQL

```sql
SELECT *
```

**Explicación:** Filtramos buscando un valor numérico específico. Algunos pasajeros (como empleados o bebés) no pagaron tarifa.

**13. Selecciona los pasajeros que viajaban solos.**

SQL

```sql
SELECT *
FROM titanic
WHERE viaja_solo = 'True';
```

**Explicación:** Filtra usando un texto que representa un valor booleano. 
📝 **Chuleta:** Aunque signifique "verdadero", en esta tabla está guardado como texto (`'True'`), por lo que necesita comillas.

**14. Muestra los pasajeros que NO sobrevivieron.**

SQL

```sql
SELECT *
FROM titanic
WHERE superviviente = 0;
```

**Explicación:** En esta tabla, la muerte está representada por un `0` numérico en la columna `superviviente`.

**15. Muestra los pasajeros que tienen menos de 10 años.**

SQL

```sql
SELECT *
FROM titanic
WHERE edad < 10;
```

**Explicación:** Operador relacional "menor que". Excluye a los que tienen exactamente 10 años. 
📝 **Chuleta:** `<` (menor), `>` (mayor), `<=` (menor o igual), `>=` (mayor o igual).

**16. Lista los pasajeros que pagaron una tarifa mayor a 50.**

SQL

```sql
SELECT *
FROM titanic
WHERE tarifa > 50;
```

**Explicación:** Operador "mayor que". Excluye a los que pagaron 50 exactos.

**17. Muestra los pasajeros que tienen 50 años o más.**

SQL

```sql
SELECT *
FROM titanic
WHERE edad >= 50;
```

**Explicación:** Operador "mayor o igual". Aquí sí incluimos a los que tienen 50 clavados.

**18. Selecciona los pasajeros que viajan con más de 2 hermanos o cónyuges.**

SQL


**Explicación:** Acota por número de familiares usando otra columna numérica.

**19. Muestra los pasajeros que no tenían ningún padre ni hijo a bordo.**

SQL

**20. Lista los pasajeros cuya clase sea distinta de 3.**

SQL

```sql
SELECT *
FROM titanic
WHERE clase <> 3;
```

**Explicación:** Operador de desigualdad. Traerá a los de 1ª y 2ª clase. 
📝 **Chuleta:** El estándar SQL para decir "distinto de" es `<>`. (Aunque sistemas como Oracle, MySQL o PostgreSQL también entienden el `!=`).

**21. Muestra los pasajeros cuya cubierta es conocida (no es nula).**

SQL

```sql
SELECT *
FROM titanic
WHERE cubierta IS NOT NULL;
```

**Explicación:** Exige que la celda tenga un valor real guardado. 
📝 **Chuleta:** Es la única manera válida de descartar los vacíos lógicos (`NULL`). Si usas `= NULL` o `<> NULL` tu consulta fallará silenciosamente y no devolverá nada.

**22. Muestra los pasajeros cuya edad desconocemos (es nula).**

SQL

```sql
SELECT *
FROM titanic
WHERE edad IS NULL;
```

**Explicación:** La cara opuesta de la anterior. Se queda exclusivamente con los registros que no tienen dato de edad.

**23. Selecciona las mujeres que viajaban en primera clase.**

SQL

```sql

```

**Explicación:** Combinamos dos condiciones usando `AND`. Ambas tienen que ser ciertas a la vez. 
📝 **Chuleta:** El `AND` hace que el embudo de resultados sea más estrecho (es restrictivo).

**Explicación:** Múltiples filtros para cruzar género y estado vital.

**33. Lista los pasajeros cuya ciudad de embarque comienza por la letra 'S'.**

SQL

```sql
SELECT *
FROM titanic
WHERE ciudad_embarque LIKE 'S%';
```

**Explicación:** Búsqueda difusa. Busca cualquier texto que empiece por una S mayúscula.
📝 **Chuleta:** El comodín `%` equivale a "cualquier cantidad de caracteres (incluso cero)".

**34. Muestra los pasajeros cuya clase literal termina en 'ird'.**

SQL

```sql
SELECT *
FROM titanic
WHERE clase_literal LIKE '%ird';
```

**Explicación:** El comodín al principio significa que no importa cómo empiece la cadena, pero debe acabar estrictamente en "ird" (por "Th**ird**").

**35. Selecciona los pasajeros que tienen una 'A' en su código de cubierta.**

SQL

```sql
SELECT *
FROM titanic
WHERE cubierta LIKE '%A%';
```

**Explicación:** Poner el comodín a ambos lados significa que busca la 'A' en cualquier posición dentro del texto (principio, medio o final).

**36. Muestra la edad de los pasajeros multiplicada por 365 (estimación en días).**

SQL

```sql
SELECT
  id,
  edad,
  edad * 365 AS edad_en_dias_aprox
FROM titanic;
```

**Explicación:** Realiza un cálculo matemático al vuelo y le asigna un nombre ficticio a la nueva columna usando un Alias (`AS`). 
📝 **Chuleta:** Las operaciones dentro del `SELECT` no alteran los datos reales de la base de datos, solo generan una proyección visual.

SQL

```sql

  id,

  padres_hijos,
  (hermanos_conyuges + padres_hijos) AS familia_total
```

**Explicación:** Suma los valores de dos columnas diferentes pertenecientes a la misma fila.

**39. Calcula la edad que tendrían los pasajeros dentro de 10 años.**

SQL

```sql
SELECT
```

**40. Muestra la clase literal en mayúsculas y la ciudad de embarque en minúsculas.**

SQL

```sql
SELECT
  UPPER(clase_literal) AS clase_literal_mayus,
  LOWER(ciudad_embarque) AS ciudad_embarque_minus
FROM titanic;
```

**Explicación:** Funciones de cadena de fila única (`Single-Row Functions`) que transforman el texto al vuelo antes de mostrarlo. 
📝 **Chuleta:** `UPPER()` pasa todo a mayúsculas, y `LOWER()` convierte todo el texto en minúsculas. Son ideales para normalizar reportes.
