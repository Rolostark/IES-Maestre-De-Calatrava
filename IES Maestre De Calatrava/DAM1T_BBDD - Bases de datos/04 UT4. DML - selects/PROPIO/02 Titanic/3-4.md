### Bloque 1: Funciones de Agregación Simples (Sin agrupar)

**Enunciado 1:** Calcula la edad promedio exacta de los pasajeros que sobrevivieron y viajaban en primera clase.

SQL

```sql
SELECT AVG(edad) AS edad_promedio
FROM titanic
WHERE superviviente = 1 AND clase = 1;
```

**Explicación:** Filtramos primero a los pasajeros con el `WHERE` y, a los que pasan el filtro, les calculamos la media matemática de su columna edad. 📝 **Chuleta:** `AVG()` ignora automáticamente los valores nulos, por lo que los pasajeros sin edad no arruinarán tu promedio.

**Enunciado 2:** Calcula cuánto dinero (tarifa) se recaudó en total de los pasajeros que embarcaron en 'Southampton' y no sobrevivieron.

SQL

```sql
SELECT SUM(tarifa) AS dinero_recaudado
FROM titanic
WHERE ciudad_embarque = 'Southampton' AND superviviente = 0;
```

**Explicación:** Acumulamos el dinero de los billetes usando `SUM()` pero limitando el cálculo a un grupo demográfico muy estricto.

**Enunciado 3:** Muestra cuál es la tarifa máxima que pagó un pasajero que viajaba solo (sin familia a bordo).

SQL

```sql
SELECT MAX(tarifa) AS tarifa_max_solitario
FROM titanic
WHERE viaja_solo = 'True';
```

**Explicación:** Escaneamos todas las tarifas buscando el "techo" de precio para este subgrupo de personas. 📝 **Chuleta:** Recuerda comprobar si tus valores lógicos están guardados como texto (`'True'`) o número (`1`), ¡las comillas dependen de eso!

**Enunciado 4:** Calcula la diferencia de edad entre el hombre más viejo y la mujer más joven a bordo.

SQL

```sql
SELECT 
    MAX(CASE WHEN genero = 'male' THEN edad END) -
    MIN(CASE WHEN genero = 'female' THEN edad END) AS diferencia_edad
FROM titanic;
```

**Explicación:** Usamos "agregación condicional". Sacamos el máximo solo fijándonos en los hombres y el mínimo solo fijándonos en las mujeres, y restamos los resultados en la misma consulta.

**Enunciado 5:** Cuenta el número total de pasajeros que tienen informada la cabina (no es nula) y cuyo puerto de embarque fue 'Cherbourg'.

SQL

```sql
SELECT COUNT(*) AS total_pasajeros
FROM titanic
WHERE cubierta IS NOT NULL AND ciudad_embarque = 'Cherbourg';
```

**Explicación:** Cuenta filas físicas tras asegurar la existencia del dato en la cubierta con `IS NOT NULL`.

**Enunciado 6:** Calcula el promedio de familiares (suma de hermanos/cónyuges y padres/hijos) que tenían los pasajeros menores de edad.

SQL

```sql
SELECT AVG(hermanos_conyuges + padres_hijos) AS promedio_familiares
FROM titanic
WHERE edad < 18;
```

**Explicación:** Suma dos columnas a nivel de fila y luego promedia el resultado de todos esos niños de forma global.

**Enunciado 7:** Muestra el porcentaje de supervivencia (promedio de la columna 'superviviente' multiplicado por 100) de todos los pasajeros.

SQL

```sql
SELECT AVG(superviviente) * 100 AS porcentaje_supervivencia
FROM titanic;
```

**Explicación:** Al ser "superviviente" una columna de `0` y `1`, su promedio nos da directamente el porcentaje en formato decimal (ej. 0.38), que pasamos a entero multiplicando por 100.

**Enunciado 8:** Calcula la tarifa media pagada por los pasajeros cuyo nombre contiene la palabra "Mrs.".

SQL

```sql
SELECT AVG(tarifa) AS tarifa_media_mrs
FROM titanic
WHERE tipo_persona LIKE '%Mrs.%';
```

**Explicación:** Hacemos una búsqueda de texto parcial para aislar a las mujeres casadas antes de promediar su billete. _(Nota: asumiendo que el dato está en `tipo_persona` según el esquema del Titanic que veníamos usando)._

**Enunciado 9:** Muestra la suma de las edades de todos los supervivientes.

SQL

```sql
SELECT SUM(edad) AS suma_total_edades
FROM titanic
WHERE superviviente = 1;
```

**Explicación:** Acumula la cifra total de todos los años vividos por las personas rescatadas.

**Enunciado 10:** Cuenta cuántos pasajeros viajaban en tercera clase y tenían más de 40 años o pagaron más de 50 de tarifa.

SQL

```sql
SELECT COUNT(*) AS total
FROM titanic
WHERE clase = 3 
  AND (edad > 40 OR tarifa > 50);
```

**Explicación:** Combinación crítica de `AND` con un bloque `OR` agrupado. 📝 **Chuleta:** **¡Paréntesis obligatorios!** Si los omites, el `AND` tiene prioridad matemática y destruiría la lógica de tu pregunta.

### Bloque 2: Agrupaciones (GROUP BY y HAVING)

**Enunciado 11:** Muestra el número total de pasajeros desglosado por clase (cuántos en primera, cuántos en segunda, etc.).

SQL

```sql
SELECT clase, COUNT(*) AS numero_pasajeros
FROM titanic
GROUP BY clase;
```

**Explicación:** "Aplasta" a todo el pasaje en 3 montones distintos según su clase y los cuenta. 📝 **Chuleta:** **Regla de oro:** Si pones `clase` suelta en el `SELECT`, TIENES que escribir `GROUP BY clase` al final.

**Enunciado 12:** Calcula la edad promedio de los hombres y de las mujeres (agrupado por género).

SQL

```sql
SELECT genero, AVG(edad) AS edad_promedio
FROM titanic
GROUP BY genero;
```

**Explicación:** Dos montones (hombres y mujeres) y una estadística matemática de cada uno.

**Enunciado 13:** Muestra el precio medio de la tarifa para cada puerto de embarque.

SQL

```sql
SELECT ciudad_embarque, AVG(tarifa) AS tarifa_promedio
FROM titanic
GROUP BY ciudad_embarque;
```

**Explicación:** Agrupa por un campo de texto para ver si un puerto era "más rico" que otro.

**Enunciado 14:** Muestra cuántos supervivientes y cuántos no supervivientes hubo en total (agrupado por la columna superviviente).

SQL

```sql
SELECT superviviente, COUNT(*) AS total
FROM titanic
GROUP BY superviviente;
```

**Explicación:** La forma más rápida de obtener un resumen de vivos vs fallecidos.

**Enunciado 15:** Calcula la tarifa máxima y la tarifa mínima que se pagó en cada una de las clases.

SQL

```sql
SELECT clase, MAX(tarifa) AS mas_cara, MIN(tarifa) AS mas_barata
FROM titanic
GROUP BY clase;
```

**Explicación:** Puedes solicitar múltiples funciones de agregación distintas apuntando al mismo agrupamiento.

**Enunciado 16:** Muestra el número de pasajeros por cada combinación de clase y género (como: Clase 1 - Mujer: X pasajeros).

SQL

```sql
SELECT clase, genero, COUNT(*) AS total_pasajeros
FROM titanic
GROUP BY clase, genero;
```

**Explicación:** Agrupación múltiple. Se generan subgrupos únicos que combinan ambas características.

**Enunciado 17:** Lista las clases que tienen una edad promedio de sus pasajeros superior a 30 años.

SQL

```sql
SELECT clase, AVG(edad) AS edad_promedio
FROM titanic
GROUP BY clase
HAVING AVG(edad) > 30;
```

**Explicación:** Filtramos los montones resultantes _después_ de haber hecho el agrupamiento. 📝 **Chuleta:** Cualquier condición basada en un resultado matemático (`SUM`, `AVG`, `COUNT`) va OBLIGATORIAMENTE en el `HAVING`, jamás en el `WHERE`.

**Enunciado 18:** Muestra las ciudades de embarque desde las que subieron más de 100 pasajeros.

SQL

```sql
SELECT ciudad_embarque, COUNT(*) AS total_embarcados
FROM titanic
GROUP BY ciudad_embarque
HAVING COUNT(*) > 100;
```

**Explicación:** Hace montones por puerto, cuenta a la gente, y tira a la papelera los puertos pequeños.

**Enunciado 19:** Calcula la tarifa promedio por clase, pero muestra solo aquellas clases donde la tarifa media sea mayor a 50.

SQL

```sql
SELECT clase, AVG(tarifa) AS tarifa_media
FROM titanic
GROUP BY clase
HAVING AVG(tarifa) > 50;
```

**Explicación:** Otro filtrado de grupo basado en el umbral económico medio.

**Enunciado 20:** Muestra el número de pasajeros por cubierta (deck), ordenado de la cubierta con más pasajeros a la que tiene menos, ignorando los que no tienen cubierta.

SQL

```sql
SELECT cubierta, COUNT(*) AS total_pasajeros
FROM titanic
WHERE cubierta IS NOT NULL
GROUP BY cubierta
ORDER BY total_pasajeros DESC;
```

**Explicación:** El flujo completo y perfecto del SQL: Filtra filas nulas (`WHERE`), luego agrupa (`GROUP BY`), luego cuenta el grupo, y por último ordena el resultado (`ORDER BY`).
