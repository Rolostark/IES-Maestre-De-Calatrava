**Enunciado 1:** Ciudades de embarque distintas (sin repetidas)

SQL

```sql
SELECT DISTINCT ciudad_embarque
FROM titanic;
```

**Explicación:** Proyecta la columna de los puertos, pero elimina todas las repeticiones para mostrarte solo los valores únicos. 📝 **Chuleta:** `DISTINCT` se coloca siempre inmediatamente después de la palabra `SELECT`.

**Enunciado 2:** Edades distintas de los que sobrevivieron, ordenadas de menor a mayor

SQL

```sql
SELECT DISTINCT edad
FROM titanic
WHERE superviviente = 1
  AND edad IS NOT NULL
ORDER BY edad ASC;
```

**Explicación:** Combina limpieza de nulos, filtrado, eliminación de duplicados y ordenación. 📝 **Chuleta:** El `ORDER BY` se ejecuta al final, por lo que ordenará perfectamente los valores únicos que saque el `DISTINCT`.

**Enunciado 3:** Pasajeros con edad desconocida (NULL) y que sobrevivieron

SQL

```sql
SELECT *
FROM titanic
WHERE edad IS NULL
  AND superviviente = 1;
```

**Explicación:** Búsqueda específica de registros con celdas vacías (`IS NULL`) combinada con un filtro de igualdad.

**Enunciado 4:** Pasajeros con cubierta asignada (no NULL) y tarifa < 20

SQL

```sql
SELECT *
FROM titanic
WHERE cubierta IS NOT NULL
  AND tarifa < 20;
```

**Explicación:** Lo opuesto al anterior. Exigimos que el dato exista (`IS NOT NULL`) y le sumamos un límite numérico.

**Enunciado 5:** Pasajeros que NO son de primera clase y embarcaron en 'Cherbourg'

SQL

```sql
SELECT *
FROM titanic
WHERE clase <> 1
  AND ciudad_embarque = 'Cherbourg';
```

**Explicación:** Operador de desigualdad `<>` (distinto de) combinado con un filtro de texto exacto.

**Enunciado 6:** Mujeres de primera clase O hombres de tercera clase (con paréntesis)

SQL

```sql
SELECT *
FROM titanic
WHERE (genero = 'female' AND clase = 1)
   OR (genero = 'male'   AND clase = 3);
```

**Explicación:** Buscamos dos perfiles completamente opuestos usando el operador `OR`. 📝 **Chuleta:** **¡Vital!** Cuando mezcles `AND` y `OR`, usa siempre paréntesis. Sin ellos, el orden matemático de SQL daría prioridad a los `AND` y la lógica de tu consulta se destruiría.

**Enunciado 7:** Pagaron > 200 O (iban en primera clase y sobrevivieron)

SQL

```sql
SELECT *
FROM titanic
WHERE tarifa > 200
   OR (clase = 1 AND superviviente = 1);
```

**Explicación:** Otra aplicación de lógica booleana. Pasarán el filtro los millonarios O los supervivientes de primera.

**Enunciado 8:** Viajaban solos (sin padres/hijos/hermanos/cónyuges) y tenían < 18 años

SQL

```sql
SELECT *
FROM titanic
WHERE padres_hijos = 0
  AND hermanos_conyuges = 0
  AND edad < 18;
```

**Explicación:** Apilamiento estricto de condiciones. Todos los `AND` deben ser ciertos a la vez.

**Enunciado 9:** Hombres > 60 O mujeres < 20

SQL

```sql
SELECT *

   OR (genero = 'female' AND edad < 20);
```

**Explicación:** Segmentación de perfiles demográficos encapsulados entre paréntesis.

**Enunciado 10:** Sobrevivieron y NO embarcaron en 'Southampton'

SQL

```sql
SELECT *
FROM titanic

  AND ciudad_embarque <> 'Southampton';
```

**Explicación:** Filtro mixto numérico y de exclusión de texto.

**Enunciado 11:** Ordenados por clase (ASC) y luego por edad (DESC) dentro de cada clase

SQL

```sql
SELECT *
FROM titanic
ORDER BY clase ASC, edad DESC NULLS LAST;
```

**Explicación:** Ordenación múltiple para resolver empates. Ordena primero por clase (1, 2, 3), y dentro de la clase 1, pone a los mayores arriba. 📝 **Chuleta:** `NULLS LAST` es oro puro. En Oracle, al ordenar de forma descendente (`DESC`), los valores nulos (desconocidos) se colocan los primeros. `NULLS LAST` los envía al fondo de la tabla.

**Enunciado 12:** Ordenados por tarifa (DESC) y, en empate, por edad (ASC)

SQL

```sql
SELECT *
FROM titanic
ORDER BY tarifa DESC, edad ASC NULLS LAST;
```

**Explicación:** Los que más pagaron van primero. Si dos pagaron 50 exactos, el desempate lo gana el más joven.

**Enunciado 13:** Los 3 importes de tarifa más distintos (top 3 tarifas distintas más altas)

SQL

```sql
SELECT DISTINCT tarifa
FROM titanic
ORDER BY tarifa DESC
FETCH FIRST 3 ROWS ONLY;
```

**Explicación:** Obtiene los 3 precios únicos más caros de todo el barco. 📝 **Chuleta:** `FETCH FIRST n ROWS ONLY` recorta la consulta para devolver solo el Top N.

**Explicación:** Primera **Función de Agregación**. Escanea todos los billetes y devuelve el valor numérico más alto.

**Enunciado 16:** Promedio de edad de los pasajeros de primera clase

**Enunciado 17:** Suma total recaudada por tickets de los pasajeros que sobrevivieron

```sql
SELECT SUM(tarifa) AS total_recaudado_sobrevivientes
```

**Enunciado 18:** Cuántos pasajeros hay en total

SQL

```sql
SELECT COUNT(*) AS total_pasajeros
FROM titanic;
```

**Explicación:** Cuenta cuántas filas físicas existen en la tabla. 📝 **Chuleta:** `COUNT(*)` lo cuenta absolutamente todo, incluyan o no incluyan datos nulos en sus celdas.

**Enunciado 19:** Cuántos pasajeros tienen informada la cubierta (no NULL)

SQL

```sql
SELECT COUNT(cubierta) AS pasajeros_con_cubierta
FROM titanic;
```

**Explicación:** Al poner una columna concreta dentro del COUNT, cambia su comportamiento. 📝 **Chuleta:** `COUNT(columna)` cuenta cuántas filas tienen un dato real en esa columna, ignorando por completo los `NULL`.

**Enunciado 24:** id, clase y coste_por_persona = tarifa / (miembros familia + 1 propio)

SQL

```sql
SELECT
  id,
  clase,
  tarifa / (hermanos_conyuges + padres_hijos + 1) AS coste_por_persona
FROM titanic;
```

**Explicación:** Realiza un cálculo de prorrateo por pasajero directamente en el `SELECT`. 📝 **Chuleta:** El `+ 1` es crítico para no romper el programa. Si alguien viaja solo (0 familiares), dividirías la tarifa entre 0 (lo cual da error matemático). Con el `+1` te cuentas a ti mismo.

**Enunciado 25:** Embarcaron en 'Queenstown' y su edad es par (MOD)

SQL

```sql
SELECT *
FROM titanic
WHERE ciudad_embarque = 'Queenstown'
  AND edad IS NOT NULL
  AND MOD(edad, 2) = 0;
```

**Explicación:** Busca edades pares usando la función módulo. 📝 **Chuleta:** `MOD(columna, 2) = 0` es la fórmula para saber si un número es par (el resto de dividir entre 2 es cero).

**Enunciado 27:** Tipos de 'clase_literal' combinados con 'genero', sin duplicados

SQL

```sql
SELECT DISTINCT clase_literal || '-' || genero AS clase_genero
FROM titanic
ORDER BY clase_genero;
```

**Explicación:** Pega el texto de dos columnas diferentes y un guion en medio (ej. "First-female"), y luego elimina los duplicados de esa nueva palabra inventada. 📝 **Chuleta:** `||` sirve para concatenar (pegar) texto en Oracle.

**Enunciado 30:** Diferencia entre la edad del más viejo y la del más joven

SQL

```
SELECT (MAX(edad) - MIN(edad)) AS diferencia_edad
FROM titanic;
```

**Explicación:** Resta el resultado de dos funciones de agregación entre sí. 📝 **Chuleta:** Puedes operar matemáticamente con el resultado de las agregaciones en el mismo `SELECT` para obtener rangos o deltas.
