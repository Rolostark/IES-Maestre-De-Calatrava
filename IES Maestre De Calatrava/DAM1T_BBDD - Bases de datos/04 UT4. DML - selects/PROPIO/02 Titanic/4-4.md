**Enunciado 1:** Calcula el precio medio de la tarifa agrupado por ciudad de embarque, pero muestra únicamente aquellas ciudades cuyo precio medio supere los 30.

SQL

```sql
SELECT ciudad_embarque, AVG(tarifa) AS precio_medio
FROM titanic
GROUP BY ciudad_embarque
HAVING AVG(tarifa) > 30;
```

**Explicación:** Agrupamos por ciudad y aplicamos el filtro estadístico en el `HAVING`. 📝 **Chuleta:** Todo filtro basado en una función (`AVG`, `SUM`, `MAX`) va siempre en el `HAVING`.

**Enunciado 2:** Muestra los datos de los pasajeros cuya cubierta es desconocida (nula), pero que sí tienen una edad registrada, pagaron una tarifa estrictamente mayor a 50 y NO sobrevivieron.

SQL

```sql
SELECT * FROM titanic
WHERE cubierta IS NULL 
  AND edad IS NOT NULL 
  AND tarifa > 50 
  AND superviviente = 0;
```

**Explicación:** Encadenamiento de filtros `AND` para buscar un perfil súper concreto con cruce de nulos.

**Enunciado 3:** Calcula la tarifa máxima y la tarifa mínima pagada, agrupando los resultados según si el pasajero sobrevivió o no, excluyendo previamente a quienes no tienen la edad registrada.

SQL

```sql
SELECT superviviente, MAX(tarifa) AS max_tarifa, MIN(tarifa) AS min_tarifa
FROM titanic
WHERE edad IS NOT NULL
GROUP BY superviviente;
```

**Explicación:** El `WHERE` limpia los datos sueltos antes de que el `GROUP BY` haga los montones.

**Enunciado 4:** Muestra las cubiertas (ignorando las desconocidas) donde la suma total de supervivientes sea mayor a 10 personas.


**Explicación:** El `WHERE` quita la cubierta nula; luego agrupa; luego el `HAVING` exige un mínimo de supervivientes.
