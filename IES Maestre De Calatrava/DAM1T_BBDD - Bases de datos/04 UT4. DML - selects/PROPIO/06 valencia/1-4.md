### Bloque 1: Explorando la Clientela y Categorías

**1. Muestra todos los campos de la tabla CATEGORIAS para ver qué secciones tiene la tienda.** SQL

SQL

```sql
SELECT * FROM categorias;
```

**Explicación:** Para una "selección total" donde queremos ver absolutamente todas las columnas sin filtrar ninguna, utilizamos el comodín asterisco (`*`).

📝 **Chuleta:** `SELECT * FROM tabla;` es la consulta más básica. Úsala siempre para explorar la estructura y los datos de una tabla nueva antes de empezar a hacer filtros complejos.

**2. Lista el nombre completo y el email de todos los clientes registrados.** SQL

SQL

```sql
SELECT nombre_completo, email 
FROM clientes;
```

**Explicación:** Aquí aplicamos una "proyección". Solo nos interesan datos específicos, así que nombramos explícitamente las columnas deseadas omitiendo el resto.

📝 **Chuleta:** Separa los nombres de las columnas con comas en el `SELECT`. El orden en el que las escribas será exactamente el orden en el que se visualizarán en el resultado final.

**3. ¿Cuántos clientes tenemos en total? (usa COUNT).** SQL

SQL

```sql
SELECT COUNT(*) AS total_clientes 
FROM clientes;
```

**Explicación:** Utilizamos una función de agregación (`COUNT(*)`) para que la base de datos cuente el número de filas totales en la tabla y nos devuelva un único número resumen.

📝 **Chuleta:** Puedes usar alias con `AS nombre_columna` para que la cabecera del resultado sea legible (por ejemplo, "total_clientes" en lugar de "COUNT(*)").

**4. Muestra los clientes que viven en la provincia de 'Barcelona'.** SQL

SQL

```sql
SELECT * FROM clientes 
WHERE provincia = 'Barcelona';
```

**Explicación:** Usamos la cláusula `WHERE` para filtrar datos. El texto entre comillas simples distingue mayúsculas y minúsculas (_Case Sensitive_), así que 'Barcelona' debe ir escrito tal cual está guardado.

📝 **Chuleta:** Si no estás seguro de cómo se guardó el dato (mayúsculas o minúsculas), puedes usar `WHERE UPPER(provincia) = 'BARCELONA'`.

**5. Obtén una lista de las provincias distintas donde tenemos clientes, sin que se repitan.** SQL

SQL

```sql
SELECT DISTINCT provincia 
FROM clientes;
```

**Explicación:** Al añadir la palabra reservada `DISTINCT` justo después del `SELECT`, Oracle elimina automáticamente todas las filas duplicadas del resultado para mostrar solo los valores únicos.

📝 **Chuleta:** `DISTINCT` afecta a todas las columnas que pongas en el `SELECT`. Si pones `DISTINCT nombre, provincia`, evaluará la combinación de ambas cosas para buscar repetidos.

**6. Muestra los nombres de los clientes que NO han facilitado un número de teléfono.** SQL

SQL

```sql
SELECT nombre_completo 
FROM clientes 
WHERE telefono IS NULL;
```

**Explicación:** En bases de datos, un valor vacío se llama `NULL`. No podemos usar el signo igual (`= NULL`) para buscarlo, la sintaxis correcta exige utilizar el operador `IS NULL`.

📝 **Chuleta:** Para buscar exactamente lo contrario (clientes que SÍ tienen teléfono), el operador sería `IS NOT NULL`.

**7. Lista los 5 clientes con más puntos de fidelidad, ordenados de mayor a menor.** SQL

SQL

```sql
SELECT nombre_completo, puntos_fidelidad 
FROM clientes 
ORDER BY puntos_fidelidad DESC 
FETCH FIRST 5 ROWS ONLY;
```

**Explicación:** Se trata de una _Top-N Query_. Primero forzamos el orden de mayor a menor con `ORDER BY ... DESC` y luego limitamos la salida a las 5 primeras filas.

📝 **Chuleta:** `FETCH FIRST n ROWS ONLY` debe ir **siempre** al final de toda la sentencia SQL, justo después del `ORDER BY`.

**8. Busca a todos los clientes que se hayan registrado durante el año 2026.** SQL

SQL

```sql
SELECT * FROM clientes 
WHERE TO_CHAR(fecha_registro, 'YYYY') = '2026';
```

**Explicación:** Convertimos la fecha de registro a texto extrayendo solo el año (`'YYYY'`) mediante la función `TO_CHAR`, y luego lo comparamos con la cadena '2026'.

📝 **Chuleta:** También puedes extraer partes de una fecha matemáticamente usando `EXTRACT(YEAR FROM fecha_registro) = 2026`.

### Bloque 2: El Inventario de Repuestos

**9. Muestra el nombre y el precio de todos los productos de la marca 'Brembo'.** SQL

SQL

```sql
SELECT nombre, precio_actual 
FROM productos 
WHERE marca = 'Brembo';
```

**Explicación:** Combinamos una proyección (para ver solo el nombre y el precio) con un filtrado simple en la cláusula `WHERE` buscando el texto exacto.

📝 **Chuleta:** Recuerda que las cadenas de texto siempre deben ir encerradas entre comillas simples (`' '`), mientras que los números van sin comillas.

**10. Lista los productos que tienen un stock crítico (menos de 10 unidades).** SQL

SQL

```sql
SELECT nombre, stock 
FROM productos 
WHERE stock < 10;
```

**Explicación:** Utilizamos el operador matemático de comparación "menor que" (`<`) en la cláusula `WHERE` sobre la columna numérica `stock`.

📝 **Chuleta:** Si quisieras incluir también los que tienen exactamente 10 unidades (10 o menos), usarías el operador "menor o igual" (`<=`).

**11. Muestra los productos cuyo precio esté entre 50 y 200 euros (usa BETWEEN).** SQL

SQL

```sql
SELECT nombre, precio_actual 
FROM productos 
WHERE precio_actual BETWEEN 50 AND 200;
```

**Explicación:** `BETWEEN` simplifica la sintaxis para buscar rangos. Hace el mismo trabajo que escribir `WHERE precio_actual >= 50 AND precio_actual <= 200`.

📝 **Chuleta:** El operador `BETWEEN` es inclusivo. Es decir, los valores límite (50 y 200 en este caso) sí entran en los resultados si existen en la tabla.

**12. Busca todos los productos que contengan la palabra 'Neumático' en su nombre.** SQL

SQL

```sql
SELECT * FROM productos 
WHERE nombre LIKE '%Neumático%';
```

**Explicación:** Usamos el operador `LIKE` para buscar patrones dentro de un texto. Los símbolos de porcentaje (`%`) actúan como comodines para indicar "cualquier texto antes o después".

📝 **Chuleta:** El comodín `%` sustituye a cero, uno o varios caracteres. El comodín `_` (guion bajo) sustituye a un único carácter exacto.

**13. ¿Cuál es el producto más caro de toda la tienda? Muestra solo su nombre y precio.** SQL

SQL

```sql
SELECT nombre, precio_actual 
FROM productos 
ORDER BY precio_actual DESC 
FETCH FIRST 1 ROW ONLY;
```

**Explicación:** Ordenamos la tabla de forma descendente (`DESC`) por el precio para que el más caro quede en primera posición, y limitamos el resultado a esa primera fila.

📝 **Chuleta:** Es una alternativa rápida a usar subconsultas como `WHERE precio_actual = (SELECT MAX(precio_actual) FROM productos)`.

**14. Calcula el stock total de piezas que hay en el almacén sumando todas las existencias.** SQL

SQL

```sql
SELECT SUM(stock) AS stock_total 
FROM productos;
```

**Explicación:** Al igual que en el ejercicio 3 usamos `COUNT` para contar, aquí usamos `SUM` para sumar matemáticamente todos los valores de la columna de stock.

📝 **Chuleta:** `SUM()` solo se puede utilizar sobre columnas que tengan tipos de datos numéricos. Si lo intentas con textos o fechas, dará error.

### Bloque 3: Pedidos y Ventas

**15. Muestra todos los pedidos que se encuentran en estado 'CANCELADO'.** SQL

SQL

```sql
SELECT * FROM pedidos 
WHERE estado = 'CANCELADO';
```

**Explicación:** Un filtrado directo buscando la cadena de texto exacta en la columna `estado`.

📝 **Chuleta:** Cuando los datos están normalizados y controlados (como tu restricción `CHECK` que fuerza las mayúsculas), las búsquedas exactas con `=` son la forma más rápida y eficiente de filtrar.

**16. Lista los pedidos realizados mediante el método de pago 'PayPal'.** SQL

SQL

```sql
SELECT * FROM pedidos 
WHERE metodo_pago = 'PayPal';
```

**Explicación:** Igual que el anterior, filtramos por coincidencia exacta asegurándonos de respetar las mayúsculas intercaladas de 'PayPal'.

📝 **Chuleta:** Si quisieras todos los pedidos MENOS los de PayPal, el operador a usar sería el de distinto: `<> 'PayPal'` o `!= 'PayPal'`.

**17. Muestra los pedidos realizados hoy mismo (9 de marzo de 2026), incluyendo la hora.** SQL

SQL

```sql
SELECT * FROM pedidos 
WHERE TRUNC(fecha_pedido) = TO_DATE('2026-03-09', 'YYYY-MM-DD');
```

**Explicación:** Como las fechas en Oracle incluyen horas, si buscas una fecha exacta sin hora no coincidirá. `TRUNC()` elimina la hora del dato original para poder compararlo solo a nivel de día.

📝 **Chuleta:** `TO_DATE` es vital para evitar errores de configuración regional. Siempre es mejor especificar el formato ('YYYY-MM-DD') que depender del que tenga el servidor por defecto.

**18. En la tabla LINEAS_PEDIDO, muestra aquellas donde se compraron más de 3 unidades de un mismo producto.** SQL

SQL

```sql
SELECT * FROM lineas_pedido 
WHERE cantidad > 3;
```

**Explicación:** Un filtro numérico sencillo con el operador "mayor que". Las filas con exactamente 3 unidades quedarán excluidas.

📝 **Chuleta:** Los operadores de comparación numéricos básicos son `=`, `>`, `<`, `>=`, `<=`, `<>`.

**19. Muestra el pedido_id, el producto_id y el subtotal de las 10 líneas de pedido más caras.** SQL

SQL

```sql
SELECT pedido_id, producto_id, subtotal 
FROM lineas_pedido 
ORDER BY subtotal DESC 
FETCH FIRST 10 ROWS ONLY;
```

**Explicación:** Proyectamos las tres columnas solicitadas, ordenamos el precio total (subtotal) de mayor a menor, y aplicamos el límite Top-N a 10 registros.

📝 **Chuleta:** Puedes ordenar e incluso filtrar basándote en columnas virtuales (como `subtotal`) exactamente igual que si fueran columnas físicas reales guardadas en el disco.

**20. ¿Qué estados distintos puede tener un pedido? (usa DISTINCT sobre la columna estado).** SQL

SQL

```sql
SELECT DISTINCT estado 
FROM pedidos;
```

**Explicación:** Al leer la columna y aplicar `DISTINCT`, Oracle colapsa todas las repeticiones ("PENDIENTE", "ENTREGADO", etc.) y nos devuelve solo el catálogo de estados que se están usando actualmente.

📝 **Chuleta:** Es una herramienta de exploración buenísima para saber qué valores reales hay introducidos en una columna categórica antes de empezar a hacer filtros complejos sobre ella.
